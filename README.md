3)Product Owner é quem define o que o cliente precisa e manda para a equipe o que deve ser feito.

 Scrum Master é quem organiza as tarefas da equipe de desenvolvimento, supervisionando qual tarefa deve ser feita, definindo prioridade.

Time de Desenvolvimento é quem faz o sistema.

4) Sprint é o desenvolvimento em etapas geralmente que duram entre 1 a 4 semanas.

Sprint planning é a parte que define o que deve ser feito, definindo o product backlog.

Daily scrum é uma reunião diária que organiza o que foi feito por cada um dos desenvolvedores para sincronizar as atividades.

Sprint Review é a parte em que se revisa tudo que foi feito ao final da sprint para receber o feedback do cliente.

Sprint Retrospective é a ultima parte da sprint no qual é visto o que pode ser melhorado ou adicionado ao sistema.

5) Product backlog é a lista de funcionalidades que o sistema precisa ter segundo o cliente.

Sprint Backlog serve para definir o objetivo da sprint para cumprir alguma funcionalidade do backlog do produto ao final dela.

Incremento é o resultado de todas as sprints, onde as funcionalidades foram implementadas ao sistema.
